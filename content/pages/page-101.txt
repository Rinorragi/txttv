VULNERABILITY ANALYSIS: Struts RCE
-------------------------------------------------

ROOT CAUSE:
Improperly validated OGNL expressions in the
Content-Type header parser allows injection of
arbitrary Java code during multipart request
processing.

EXPLOIT VECTOR:
POST /upload HTTP/1.1
Content-Type: %{(#_='multipart/form-data')
.(#cmd='calc').(#iswin=(@java.lang.System@
getProperty('os.name').toLowerCase().contains
('win')))...}

PAYLOAD CHARACTERISTICS:
• No authentication required
• Works through reverse proxies
• Bypasses basic WAF regex filters
• Can execute OS commands directly

DETECTION INDICATORS:
→ Unusual Content-Type headers (>200 chars)
→ OGNL expression patterns: %{, #, @java
→ Outbound connections to unexpected IPs
→ New processes spawned by Java

PATCH DETAILS:
Struts 2.6.5 validates Content-Type against
strict allowlist before OGNL evaluation.
